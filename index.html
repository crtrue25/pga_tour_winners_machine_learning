PGA Tour Data (2010-2018)¶

Predicting whether a player won on the PGA Tour in a given year¶

The goal of this project is to find out what factors go into winning an event on the PGA Tour. Many good players each year fail to win an event on the PGA Tour. Using data from the PGA Tour, I am going to attempt to create a model that can predict whether a player won an event or not.

Dataset Breakdown¶

  • Player Name
  • Year
  • Rounds: Competitive rounds played on the PGA Tour for the year
  • Fairway Percentage: The percentage of tee shots on Par 4s and 5s that land in the fairway
  • Avg Distance: The average distance of a tee shot on Par 4s and 5s.
  • gir: The percentage of greens hit in regulation. Reaching the green in 2 shots below the par
  • Average Putts: The average number of putts per round for the given player and year
  • Average Scrambling: The percentage of times when a player makes par after missing the green in regulation
  • Average Score: The average score for a round
  • Points: Total FedEx Cup Points earned for the given season
  • Wins: Amount of wins on the PGA Tour for that player in a season
  • Top 10: Amount of times where a player finishes inside the Top 10 in a tournament
  • Money: Amount of money the player earned in that season

Strokes Gained: A metric used on the PGA Tour to measure a players performance on each shot

  • Average SG Putts: Average strokes gained on all shots taken from the green
  • SG:OTT: Average strokes gained on all shots taken from the tee
  • SG:APR: Average strokes gained on all shots taken outside of 50 yards (excluding tee shots)
  • SG:ARG: Average strokes gained on all shots taken inside 50 yards (excluding the green)
  • Average SG Total: Average strokes gained on all shots

Replace NaNs, Change Dtypes, Clean Values¶

Wins and Top 10s:

  • In the previous step, I found that there are 1395 nulls and 325 nulls respecitively, between these two variables, so I removed those with fillna
  • And before that, we noticed that Wins and Top 10s are marked as a float dtype. We need these to be int because you can't have partial Wins or Top 10s.

Points:

  • By looking at the preview of the dataset above we can see the values are formatted like the following: 1,006. Python doesn't like that, so we can remove the commas using apply alongside the lambda function. For each value that has a comma in it, that comma will essentially, be deleted.
  • Now we can change the dtype from object to int

Money:

  • Similar to Points, the money field is formatted like the following: $2,680,487. We can fix that by using apply and lambda.
  • Finally we can change this dtype from object to float.
In [92]:
#Replace NaNs and make int
df['Wins'].fillna(0, inplace = True)
df['Wins'] = df['Wins'].astype(int)

df['Top 10'].fillna(0,inplace = True)

#Remove comma in points and make int
df['Points'] = df['Points'].apply(lambda x: x.replace(',', ''))
df['Points'] = df['Points'].astype(int)

#Remove $ and commas in money and make int
df['Money'] = df['Money'].apply(lambda x: x.replace("$", ''))
df['Money'] = df['Money'].apply(lambda x: x.replace(',', ''))
df['Money'] = df['Money'].astype(float)
